# CRM SaaS LP作成用プロンプト

あなたはAppleで働く世界トップレベルのWebデザイナー兼フロントエンドエンジニア、そして優秀なコピーライターです。その卓越した感性と技術力を活かし、架空のCRM SaaS「CustomerFlow」のランディングページ（LP）を構築してください。

## 1. デザインコンセプト：Appleの美学（引き算と没入感）

- **コアコンセプト**: 「引き算の美学」。ノイズを極限まで削ぎ落とし、主役（ソフトウェアの価値）を劇的に際立たせる。
- **カラーパレット**:
- **プライマリーカラー**: `#DC143C` (情熱的で洗練された赤)。
- **背景**: `#FFFFFF` (純白)。
- **テキスト**: `#000000` (黒) と `#666666` (グレー) の使い分け。
- **タイポグラフィ**:
- **劇的なジャンプ率**: 画面からはみ出るほどの巨大な見出し（Hero Text）と、繊細で小さな本文の対比。
- フォントは `Inter` と `Noto Sans JP` を使用。LP内の文言は、見出しやナビゲーションを含め全て日本語にする。日本語の美しさを引き出すタイポグラフィを意識すること。
- **レイアウト & モーション**:
- **Framer Motion (必須)**: スクロールに合わせて要素が浮き上がる、ダッシュボード画面がパララックスで動くなど「物語を語るインタラクション」を実装。
- **広大な余白**: コンテンツ間に息を呑むほどの余白（ブリージング・ルーム）を設ける。

## 2. LPの構成（BtoB SaaSの王道構成）

以下の構成要素を使用し、単なる情報の羅列ではなく、一つの物語として流れるように配置してください。

### Section 1: ファーストビュー（ヒーローエリア）
- **要素**: 【利用シーン想定 / メリット紹介】
- **演出**:
- 画面一杯に実際のダッシュボード画面（モックアップ）を配置し、ツールの使用イメージを直感的に伝える。
- 「顧客との関係を自動化し、売上を最大化する」といった明確なメリットをヘッドラインで訴求。
- CTAボタン（「無料で始める」など）を目立つ位置に配置。

### Section 2: ロゴ掲載エリア
- **要素**: 【パートナー企業紹介 / 導入事例紹介】
- **演出**:
- 有名企業やプラットフォーム（Salesforce、HubSpot、Slackなど）のロゴを横並びで配置。
- 「信頼される企業が使っている」という権威付けを行う。

### Section 3: よくあるお悩み
- **要素**: 【よくあるお悩み / ギャップ】
- **演出**:
- 「顧客管理が煩雑で、営業チームが疲弊していませんか？」といった共感を呼ぶ問いかけ。
- 現状の課題を明確にし、「でも、それは変えられます」とギャップを提示。

### Section 4: 機能紹介エリア
- **要素**: 【具体的なサービス内容の紹介】
- **演出**:
- 主要機能（顧客管理、自動フォローアップ、分析ダッシュボードなど）をアイコンと短い説明文で紹介。
- 各機能に対応するUI画面のスクリーンショットやアニメーションを配置。
- グリッドレイアウトで整理し、視認性を高める。

### Section 5: ステップ紹介
- **要素**: 【具体的なサービス内容の紹介（利用フロー）】
- **演出**:
- 「01」「02」「03」と番号を振り、導入までの簡単なステップを視覚的に表現。
- 「たった3ステップで始められる」という手軽さを強調。

### Section 6: メリット訴求エリア
- **要素**: 【選ばれる理由 / メリット紹介】
- **演出**:
- 「なぜCustomerFlowが選ばれるのか」を4つ程度のボックスで説明。
- 各ボックスには、具体的なメリット（「時間を50%削減」「売上20%向上」など）を数値で示す。

### Section 7: 連携ツール紹介
- **要素**: 【パートナー企業紹介】
- **演出**:
- 既存のツール（Gmail、Slack、Zapierなど）との連携を示すロゴ一覧。
- 「今使っているツールとシームレスに連携」というメッセージで導入ハードルを下げる。

### Section 8: 料金プラン
- **要素**: 【スペック紹介】
- **演出**:
- Free、Pro、Enterpriseなどのプランを表形式で比較。
- 各プランの価格、機能差分を明確に提示。
- 推奨プランを視覚的に強調（例：「Most Popular」バッジ）。

### Section 9: FAQ
- **要素**: 【よくある質問】
- **演出**:
- アコーディオンUIで質問を収納し、クリックで展開する形式。
- 「セキュリティは大丈夫？」「導入にどれくらい時間がかかる？」など、典型的な疑問に回答。

### Section 10: クロージング
- **要素**: 【メリット紹介（最終訴求）】
- **演出**:
- 「今すぐ顧客管理を変革しよう」といった行動を促すメッセージ。
- 大きなCTAボタン（「無料で始める」）を配置し、コンバージョンを促す。

## 3. 技術スタックと要件

- **Framework**: React (Vite) + TypeScript
- **Styling**: Tailwind CSS (設定で colors や fontFamily を拡張すること)
- **Animation**: framer-motion (useScroll, useTransform, AnimatePresence などを駆使)
- **Icons**: lucide-react
- **Images**: 画像はまずUnsplashから curl コマンドで取得する指示を含めること（キーワード: modern office, dashboard ui, business team, technology）。Unsplashから取得できない、または正しく表示されない場合に限り、Nano Bananaを使用して画像を生成すること。

## 4. 成果物

上記要件を満たす `src/App.tsx`, `tailwind.config.js`, `index.css` のコード、および必要な画像アセットを生成・配置してください。コードはコピペで動作するようにし、各セクションには対応する構成要素のコメント（例：`{/* ファーストビュー - 利用シーン想定 / メリット紹介 */}`）を入れてください。
